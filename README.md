# Pool Simulator

Десктопное приложение на **PySide6** для **Monte Carlo**-симуляций покерного стейкинг-пула в спин-энд-гоу (Spin & Go, Expresso, Nitro и т.д.). Поддерживается два верхнеуровневых режима симуляции: **Swing** (дисперсия относительно EV) и **EV** (распределение ожидаемой прибыли по выбранной формуле).

## Что это дает бизнесу и менеджменту

**Pool Simulator** помогает принимать решения по управлению покерным пулом на основе вероятностей, а не интуиции.

- Оценивает диапазон рисков  для заданного объема турниров.
- Сравнивает EV-модели выплат и их влияние на ожидаемый финансовый результат.
- Позволяет быстро проверять сценарии по лимитам, рулам, составу пула и структуре множителей.
- Поддерживает портфельный режим (несколько конфигураций одновременно), что удобно для планирования пула целиком.
- Доступен как desktop-приложение и как Windows-установщик для удобной передачи non-technical пользователям.

## Обзор

### Источники случайности (на один турнир)

1. **Множитель** — какой призовой множитель выпал (категориальное распределение по структуре).
2. **ChipEV** — ожидаемая чип-результативность игрока (эмпирическое распределение по размеру стека).
3. **All-in luck** — отклонение фактических фишек от ожидаемых (условное распределение на ChipEV).

Движок использует **батчи с мультиномиальным распределением** по множителям и векторизованную выборку ChipEV и алл-ина, а не пошаговую симуляцию каждого турнира по отдельности (в режиме **Precise** для Swing).

### Два режима симуляции (переключатель **Target** вверху блока параметров)

| Режим | Что моделируется | Назначение |
|--------|------------------|------------|
| **Swing** | \( \text{Result} - \text{EV} \) по выбранной формуле | Дисперсия «свинга» относительно выбранного EV. |
| **EV** | Суммарный **EV** пула за \(N\) турниров по выбранной формуле | Распределение ожидаемой прибыли при тех же случайностях (и центрировании ChipEV), без вычитания Result. |

Аналитические **σ** (в панели параметров) рассчитываются **только в режиме Swing**; в режиме EV поля аналитической сигмы скрыты (показ «—»).

---

## Режимы точности симуляции (**Mode**)

### Режим **Swing** (без изменений семантики)

- **Eco** — быстрая CLT-аппроксимация для свинга; для неаддитивных EV-формул используется прокси через AvgEV (как и ранее в коде).
- **No ChipEV Var** — CLT-свинг при **фиксированном** ChipEV (без дисперсии ChipEV, остаётся только алл-ин шум там, где он учитывается).
- **Precise** — полная поштучная выборка ChipEV и алл-ина (мультитред для тяжёлых расчётов).

### Режим **EV**

- **Precise** — для каждого турнира: множитель → ChipEV из сдвинутого распределения под **целевое среднее для данной строки множителя** → для формул, где нужен полный пул, считается суммарный EV по тому же пайплайну, что и для Swing-ветки.
- **Eco** — сэмплируются только **количества по множителям**; для аддитивных формул EV собирается через CLT по сумме ChipEV по ярусам; для **AW EV** и **AvgEV4** — симулируются **средние ChipEV по множителям** и затем применяется та же логика, что в `ev_formulas.py`.
- **No ChipEV Var** — только случайность множителей; ChipEV **строго равен** заданным в таблице значениям по каждому множителю.

## Формулы EV

Реализация в `pool_simulator/core/ev_formulas.py`:

- **AvgEV**, **MultEV**, **MultEV50**, **MultEV4** — аддитивные (EV по турниру суммируется).
- **AW EV**, **AvgEV4** — неаддитивные (зависят от агрегатов по всей выборке турниров).

Свинг всегда: \( \text{Swing} = \sum \text{Result} - \text{EV}_{\text{formula}} \).

---

## ChipEV: правило приоритета (важно для UI)

- **Истина для симуляции** — колонка **ChipEV** в таблице **Multiplier Participation** (значение **на каждую строку множителя**).
- Поле **Mean ChipEV** — массовый ввод: заполняет/масштабирует ChipEV в таблице (в т.ч. при переменных стеках). После ручной правки таблицы в расчёт идут **только** значения из таблицы.

Для структур, где у нескольких множителей **одинаковый стек**, словарь `{stack: chip_ev}` был бы неоднозначен; в режиме **EV** (и где нужно) в движок передаётся список **per-multiplier** ChipEV.

## Валюта (EUR → USD)

Для румов в **EUR** (Betclic, Winamax, iPoker и др. по правилам в `models.py`) результаты одиночной симуляции в интерфейсе **переводятся в USD** умножением на **EUR/USD** из панели. Средние и графики в `$` отражают эту конверсию; **ожидания в «нативных» BI** без курса набирай отдельно или ставь курс **1.0** для проверки.

---

## Параметры симуляции

- **Tournaments (N)** — число турниров в одном «прогоне».
- **Simulations (M)** — число независимых прогонов; доступны пресеты до **100M** (в т.ч. 50M, 100M) в одиночном режиме и портфеле.
- **EV Formula** — одна из перечисленных в `EVFormula`.
- **Chart metric** в результатах: **BI**, **BI/10k**, **$** (для EUR-рума в режиме `$` используется конвертированная выдача).

---

## Результаты и таблица

- **Swing**: сводная таблица как раньше (конфигурация, комната, стейк, формула, \(N\), σ в $ и BI и т.д.).
- **EV** (и при необходимости другие метрики с `metric_label != Swing`): дополнительная колонка **Mean ($)** — среднее по симуляциям в валюте отображения.

Экспорт CSV/XLSX и гистограммы/перцентили соответствуют текущей метрике (`metric` в метаданных экспорта распределения).

---

## Экспорт распределения (CSV)

Файл с разделителем **`;`**, кодировка **UTF-8-SIG**:

1. Секция **meta_key / meta_value** (`chart_mode`, `n_tourneys`, `n_simulations`, `metric`, …).
2. Секция **percentile / value**.
3. Одна колонка сырых значений: заголовок может быть **`swing`**, **`ev`** или **`result`** в зависимости от режима.

Универсальный разбор в аналитике: искать секцию данных по имени `swing` | `ev` | `result`.

---

## Pool Mode (портфель)

Панель **Pool Mode (Portfolio)**: несколько конфигураций (сетап + participation + ChipEV + формула + \(N\)), общий прогон с суммированием метрик по симуляциям. Режим **Target** (Swing / EV) синхронизируется с переключателем в основной панели симуляции. Для **EV** в записи портфеля сохраняется список ChipEV по множителям для корректных прогонов.

Импорт из Excel, сохранение/загрузка профилей — см. `pool_importer.py`.
